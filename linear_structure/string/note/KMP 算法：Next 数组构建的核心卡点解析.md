### 一、先彻底搞懂：i和j到底怎么指向前后缀？

首先纠正你一个小误解：**i不是指向第i+1个字符，i就是模式串的下标（从0开始），对应子串是 ** **`pattern[0...i]`** **（前i+1个字符）**；j的核心作用有两个（这是关键！）：

1. **数值含义**：j = 「`pattern[0...i]` 的最长相等真前缀/真后缀的长度」；

2. **指针含义**：j 同时是「当前匹配的前缀的末尾下标」（前缀是 `pattern[0...j-1]`，长度正好是j）；

而后缀的末尾下标就是i（后缀是 `pattern[i-j+1...i]`，长度也正好是j）。

#### 用一个直观例子讲透（模式串 `pattern = "abab"`，下标0-3）

我们的目标是为每个i（0-3）计算next[i]，核心是：**用j表示“前缀末尾”，i表示“后缀末尾”，比较这两个位置的字符是否相等**。

|步骤|i（后缀末尾）|子串pattern[0...i]|j（前缀末尾/长度）|核心对比（前缀末尾 vs 后缀末尾）|next[i]|
|---|---|---|---|---|---|
|1|0|"a"|0|单个字符无前后缀，直接设0|0|
|2|1|"ab"|0|前缀末尾j=0（字符'a'） vs 后缀末尾i=1（字符'b'）→ 不相等|0|
|3|2|"aba"|0|先对比j=0（'a'） vs i=2（'a'）→ 相等|j=1，next[2]=1|
|4|3|"abab"|1|对比j=1（'b'） vs i=3（'b'）→ 相等|j=2，next[3]=2|
#### 可视化拆解（关键！）

以i=2（子串"aba"）为例：

- 我们要找"aba"的最长相等真前缀/真后缀：

    - 真前缀可选："a"（长度1）、"ab"（长度2）；

    - 真后缀可选："a"（长度1）、"ba"（长度2）；

    - 最长相等的是长度1（前缀"a"，后缀"a"）。

- 此时j=1（数值含义：长度1）；j=1作为指针，指向前缀的末尾？不——j=1表示「前缀是pattern[0...0]（长度1）」，后缀是pattern[2...2]（长度1）；

👉 简单说：**j表示“前缀的长度”，前缀就是前j个字符（pattern[0]到pattern[j-1]）；后缀是后j个字符（pattern[i-j+1]到pattern[i]）**。

再以i=3（子串"abab"）为例：

- j=1（上一步的结果），此时对比 pattern[j=1]（'b'） 和 pattern[i=3]（'b'）→ 相等；

- j变成2（长度2），意味着：前缀是pattern[0...1]（"ab"），后缀是pattern[2...3]（"ab"），正好相等，长度2 → next[3]=2。

#### 核心结论：i和j的关系

- **i**：遍历模式串，每个i对应“要计算next值的子串的末尾”（后缀的末尾）；

- **j**：始终指向“当前候选的前缀的末尾”（或者说“当前候选的相等前后缀长度”）；

- 我们做的所有操作，本质是：**用j的回溯，快速找到“能和i位置匹配的前缀末尾”，而不用从头比较**。

### 二、为什么暴力法是O(n²)，而这个算法是O(n)？

先讲清楚两种算法的核心区别，你就明白时间复杂度的差异了。

#### 1. 暴力法（双层循环，O(n²)）：对每个i，都“从头试”

暴力法的思路很直白：

- 对每个i（0到n-1），要找 `pattern[0...i]` 的最长相等前后缀；

- 从「最长可能的长度」（比如i，因为真前后缀最长是i）开始，逐个缩短长度，直到找到相等的；

- 比如i=3（子串"abab"）：

    - 先试长度3：前缀"aba" vs 后缀"bab" → 不相等；

    - 再试长度2：前缀"ab" vs 后缀"ab" → 相等，停止；

    - 这个过程要比较2次。

**时间复杂度为什么是O(n²)？**

- 对每个i，最多要比较i次（比如pattern="abcd"，每个i都要试到长度0）；

- 总次数是 0+1+2+...+(n-1) = n(n-1)/2 → 时间复杂度O(n²)。

#### 2. KMP构建next的算法（O(n)）：i只前进，j只回溯（不重复比较）

这个算法的核心优化是：**利用已经算好的next值，跳过所有“必然不匹配”的比较，让i只递增、j的回溯次数总和不超过n**。

##### 关键前提：

- **i只增不减**：i从1到n-1，只会往后走，从不回头（一共走n-1步）；

- **j的回溯次数有限**：j虽然会回溯（比如j从3回到1），但j的**总回溯次数**不会超过i的递增次数（因为j最多增长到n，回溯时减少的次数，永远补不上之前增长的次数）。

##### 用例子看“无重复比较”（pattern="abac"）：

暴力法算i=3（子串"abac"）：

- 试长度3：前缀"aba" vs 后缀"bac" → 不相等；

- 试长度2：前缀"ab" vs 后缀"ac" → 不相等；

- 试长度1：前缀"a" vs 后缀"c" → 不相等；

- 共比较3次。

KMP算法算i=3：

- 上一步i=2时，j=1（next[2]=1）；

- 先对比pattern[j=1]（'b'）和pattern[i=3]（'c'）→ 不相等；

- 直接回溯j到next[j-1]=next[0]=0（利用已算好的next值，跳过“试长度2、3”的步骤）；

- 再对比pattern[j=0]（'a'）和pattern[i=3]（'c'）→ 不相等；

- 共比较2次，且没有重复比较。

##### 时间复杂度的数学本质：

把整个算法的操作次数拆成两部分：

1. i递增的次数：n-1次（从1到n-1）；

2. j的操作次数：j要么递增（最多n次），要么回溯（每次回溯j减少，但总回溯次数≤n）；

总操作次数 = i的递增次数 + j的增减次数 ≤ n + n = 2n → 时间复杂度O(n)。

简单说：**暴力法是“每个i都从头试”，而KMP是“用之前的结果省步骤”，让所有操作都只走一遍，没有重复**。

### 再用一个完整例子串起来（帮你巩固）

模式串 `pattern = "abab"`（n=4），完整走一遍next构建：

1. 初始化：next[0]=0，i=1，j=0；

2. i=1（字符'b'）：

    - pattern[1]（'b'）≠ pattern[0]（'a'）→ j=0 → next[1]=0，i=2；

3. i=2（字符'a'）：

    - pattern[2]（'a'）= pattern[0]（'a'）→ j=1，next[2]=1，i=3；

4. i=3（字符'b'）：

    - pattern[3]（'b'）= pattern[1]（'b'）→ j=2，next[3]=2，i=4（循环结束）；

最终next=[0,0,1,2]。

- 全程i只走了3步（1→2→3）；

- j只走了2次递增（0→1→2），0次回溯；

- 总操作数=3+2=5 ≤ 2*4=8 → 完全符合O(n)。

### 总结

1. **i/j与前后缀的关系**：

    - i是「当前子串的末尾下标」（后缀末尾）；

    - j既是「最长相等前后缀的长度」，也是「前缀的末尾下标+1」（前缀是pattern[0...j-1]）；

    - 比较pattern[j]和pattern[i]，本质是比较“候选前缀的末尾”和“候选后缀的末尾”。

2. **时间复杂度从O(n²)到O(n)的核心**：

    - 暴力法：每个i都从头试，重复比较；

    - KMP法：i只前进不回头，j利用已算的next值回溯（无重复比较），总操作数是线性的。
