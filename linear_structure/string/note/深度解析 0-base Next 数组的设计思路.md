### 1. 0-base next数组的**标准定义**（精准无歧义）

对于0-base的模式串 `pattern`（下标范围 `0 ~ len-1`），`next[i]` 表示：

模式串的子串 `pattern[0...i]`（从第0个字符到第i个字符，共i+1个字符）中，**最长的、相等的「真前缀」和「真后缀」的长度**。

### 2. next数组的**核心目的**

next数组的设计，本质是为KMP匹配阶段提供「**最短的无效回溯路径**」——

暴力匹配模式串时，一旦某个位置不匹配，主串指针要回溯到“匹配起始位置+1”，模式串指针回到0，大量重复比较；

而next数组记录了模式串自身的“前后缀重复信息”，匹配失败时，**模式串指针不用回到0，而是回溯到next[k-1]（k是当前不匹配的位置）**，主串指针完全不回溯，这是KMP比暴力法快的核心原因。

一句话总结目的：**用模式串自身的前后缀信息，消除匹配时的重复比较，将匹配的时间复杂度从O(n*m)降到O(n+m)**（n是主串长度，m是模式串长度）。

暴力法计算next数组（找最长相等前后缀）时，对每个i，要从“最长可能的长度”开始，逐个缩短到0，重复比较，时间复杂度O(n²)——痛点是“重复比较”。

为了复用信息，设计j同时表示“当前重合长度”和“前缀末尾下标+1”——这样可以用j的移动，直接对比“前缀的下一个字符”和“后缀的下一个字符”，不用重新截取子串。

当pattern[i] != pattern[j]时，j不能直接归0，而是回溯到`next[j-1]`——因为`next[j-1]`是`pattern[0...j-1]`的最长重合长度，这是“最短的回溯路径”，既保证不遗漏，又保证不重复。

- `next[0] = 0`：单个字符无前后缀，是所有推导的基础；

- j=0时直接设next[i]=0：避免j=-1导致数组越界，也是“无法再回溯”的兜底逻辑。
