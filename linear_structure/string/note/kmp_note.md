# KMP算法
## 核心逻辑
**匹配失败→主串不回溯，只让模式串回溯**
1.  ### BF算法的局限
    ```c
    int indexBF(const String *s, const String *t) {         //Brute Force  O(n*m)
        int i = 0, j = 0;
        while (i < s->length && j < t->length) {
            if (s->data[i] == t->data[j]) {
                i++;
                j++;
            } else {
                i = i - j + 1;  //从第i开始匹配，匹配到第j个后失败，则退回到 (i + j) - j 再取下一位 
                j = 0;
            }
        }
        if (j == t->length) {
            return i - j;
        }
        return -1;
    }

    ```
    当匹配失败时，主串和模式串都回到起始位置（i = i - j -1 , j = 0）,从头重新匹配
2. ### KMP算法
     **主串不回溯，只让模式串回溯到最佳位置（j = next[j] ）**
    ## 逻辑链
    - BF 的问题：匹配失败→主串回溯，重复比对已匹配的部分；
    - KMP 的思路：匹配失败→主串不回溯，只让模式串回溯；
    - 模式串该回溯到哪？→ 找 “已匹配部分的最长公共前后缀”，其长度就是最优回溯位置（因为这部分已经匹配过，不用再比）；
    - 为了不用每次匹配失败都临时算最长公共前后缀 → 提前预处理模式串，把 “每个位置 j 失败时的最长公共前后缀长度” 存在 next [j] 里；
    - 匹配时，只要失败，就查 next [j]，让模式串回退到这个位置，主串继续往后走 → 彻底避免重复比对，实现 O (n+m) 的时间复杂度。

    1. **模式串匹配失败时的最优回退位置 = 已匹配部分的最长公共前后缀长度**
    2. **next 数组的数值，本质是 “模式串每个位置 j 匹配失败时，其前 j 个字符的最长公共前后缀长度” —— 把 “长度” 直接作为 “回退位置”，因为长度对应的就是前缀的末尾下标（比如长度 2 → 回退到 j=2**
3. ### next 数组
    **Q: 什么叫最长公共前后缀长度？为什么它能决定KMP算法回溯的位置**
    **A:**
       1. 最长公共前后缀长度，本质是「模式串已匹配部分中，能复用的前缀长度」；
       2. 这个长度决定回溯位置的原因：
        - 已匹配部分的 “最长公共后缀” = 主串的某段字符；
        - 该后缀又 = 模式串的 “最长公共前缀”；
        - 所以模式串回退到这个长度的位置，就能直接复用这段已匹配的字符，不用从头比对；
       3. KMP 的优化本质：把 “主串无意义回溯” 换成 “模式串基于最长公共前后缀的精准回溯”，而这个 “精准位置” 就是最长公共前后缀的长度。 
    **你可以记住这个比喻**
    - 模式串像一把钥匙，主串像锁孔；
    - BF 是钥匙插错了就全拔出来重新插；
    - KMP 是钥匙插错了，只拔到 “已经插对的最长部分”（最长公共前后缀长度），再继续插 —— 而 “最长公共前后缀长度” 就是 “不用拔出来的那段钥匙的长度”，也就是回溯后要停的位置。
