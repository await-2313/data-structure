# 数据结构C语言手写代码学习计划（8周，每日2-3小时）
## 核心原则
1. 手写优先：先闭卷手写代码，再对照修正，最后编译运行验证；
2. 从易到难：线性结构 → 树形结构 → 哈希/排序 → 综合巩固；
3. 每日验收：完成任务后必须手写代码并运行通过，才算达标；
4. 理解为本：不死记代码，重点理解“为什么这么写”（如二级指针、循环队列判满）。

## 额外建议
1. 工具准备：使用VSCode/Dev-C++，编译时开启警告（如GCC加-Wall），及时发现内存泄漏、野指针；
2. 错题本：建立文档，记录每次手写的错误（如“BST删除双孩子节点漏释放内存”），标注原因和修正方案；
3. 节奏调整：若某模块难度大，可延长1-2天学习，保证理解透彻，不赶进度；
4. 后续过渡：第8周后可逐步学习C++，用C++类封装这些数据结构（如BST类），为算法竞赛/就业做准备。
